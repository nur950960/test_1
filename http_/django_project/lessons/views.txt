View - это функция, которая принимает запром и возвращает ответ 

Если в drf  мы пишем views через функции, то их нужно оборачивать в декоратор 'api_view' (rest_framework.decorators.api_view), в который вы передаете список с методами, которые обрабатываются функцией 

Если  в url есть динамичный элемент, то функция так же принимает вторым аргументом его (например id)

View должна возвращать Response или любой класс из 'django.http'

APIView - класс, который позоваляет вам на классах писать  view 

тот метод, который вы переопределите в классе, тот и будет обрабатывать ваш view ('get', 'post', 'put', 'patch', 'delete')
