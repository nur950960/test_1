Git - это распределенная система контроля версий. Преднозначена для отслеживания и ведения истории изменения файлов, в вашем проекте 

Github - это сайт куда мы можем отправлять все что находится на нашем ноутбуке, что бы другие люди могли посмотреть на мой проект и стянуть (скачать) к себе 

1.git init - что бы создать скрытую директорию (.git), инициализация 
2.git add . - просматривает изменения в нашем проекте, и добовляет в промежуточное пространство (index)
3.git commit -m 'message' - фиксирует изменения из индекса в локальный репозиторий 
4.git remote add {название связки(origin)} - связывает локальный и удаленный репозиторий 
5.git push {название связки(origin)} master либо main - отправляет данные из локального в удаленный репозиторий
6.git pull origin master либо main - отправляет данные из удаленного в локальный репозиторий


Закидываем изменения в готовый репозиторий 
1.git add git_github.txt (название файла)
2.git commit -m 'update file - git_github.txt' 
3.git push origin master либо main 

Стягиваем изменения из удаленного репозитория 
1.git clone git@github.com:nur950960/test_1.git 
2.git pull origin master 

branch - ветка (версия проекта), нужня для того что бы разрабатывать свой функционал отдельно от других разработчиков 

локальная ветка, удаленная ветка, вы можете связать их(работая с git pull  git push)

git branch - показывает список векток 
git branch {name} - cоздание ветки (name) 
git checkout {name} - переключения между ветками 
git checkout -b {new_branch} {commit_key}
git merge name1 name2 - соединяет ветки (name1 и name2)
git marge name1 - соединяет ветки  
git log - истрория коммитов 

# Зафиксированное(committed) - файл уже сохранен в вашей локальной бд
# Измененное (modified)- файлы которые поменялись но не зафиксированны
# Подготовленное (staged) - измененные файлы отмеченные для включения в след коммит
